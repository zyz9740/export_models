# OpenVINO benchmark_app Usage

## GPU test

```bash
cd benchmark
benchmark_app -m ../converter/pointnet_sem_seg_simplified.xml -d GPU -hint latency -infer_precision f16
```

## CPU test

```bash
cd benchmark
benchmark_app -m ../converter/pointnet_sem_seg_simplified.xml -d CPU -hint latency
```

`-infer_precision f16` on `-d CPU` fails on this machine's OpenVINO build
with `RuntimeError: object is not initialized` (CPU plugin does not support
the explicit FP16 inference-precision hint here) -- so the CPU log is
captured at the CPU plugin's default precision instead. GPU supports the
hint and is run at FP16 as required.

## Notes

- Input shape is fixed to `[1,9,4096]` (static) by the exported IR -- a
  fully dynamic shape produces incorrect GPU output, see
  `../validation/validation_report.md`.
- `benchmark_cpu_result.txt` and `benchmark_gpu_result.txt` are raw logs generated by `benchmark_app`, captured against the static-shape IR.
